< 02. 프로젝트 분석 >

[ 09. 프로젝트 분석 ]
	1. 분석 단계
		1) AS-IS
			- 프로젝트 목적 및 배경
			- 현업과 고객의 요구사항
		2) 시장흐름과 경쟁사
			- 사용자 & 서비스 대상
			- 인프라와 사용자 환경
		3) 벤치마킹과 시사점
			- 트렌드와 아이덴티티
			- 서비스와 시스템의 개선점
	
	2. 분석 목적 & 배경
		1) Inside
			ex. 주력 사업의 확장, ...
		2) Outside
			ex. 관련 법안 발의 및 변경 대응, ...
	
	>> 프로젝트 "목적 → 목표 → 전략 → 전술" 시나리오를 만들어라
		ex. 목적		|목표			|전략					|전술
			매출 증대	|조직 개편		|브랜드 인지도 개선 		|실행안1, ...
					|사이트 리뉴얼	|가입/구매&UX/UI 개선	|실행안1, ...
					|해외 런칭		|개인화 서비스 강화		|실행안1, ...

[ 10. As Is 분석 ]
	- 현재의 정보구조와 화면 UI 설계는 어떻게 이루어져 있으며, 서비스에 요구되는 기능은 무엇이 있는가
	
	1.	정보 구조(IA) 분석
		- 구조 파악의 이유를 명확히 하고 분석 계획을 세운다
		1) 계층 구조
			- 요구사항을 기준한 TO-BE 개선 방안의 초안을 설계할 수 있다
		2) 접근 권한
			- 해당 페이지에 접근할 수 있는 권한 조건 파악
		3) 관리자 페이지
		4) 페이지 타입
			- 기능 없는 static한 페이지 (CEO 정보 등)
			- 프로그래밍 필요한 페이지 (인풋과 아웃풋이 출력되는지)
			- 링크 페이지
		5) 업데이트 주기
		6) 운영인프라
			
	2. UX/UI
	3. function

[ 11. 프로젝트 타겟 분석 ]
	1. 타겟, 누구를 위한 서비스인가
		1) WHO : 누가 방문하는가
		2) WHY : 왜 방문하는가 (무엇을 얻고자 하는가)
		3) WHAT : 무엇을 전달 할 것인가
	
	2. 타겟 설정
		- User case 구분
		+ 로그 분석 (GA 등)

[ 12. 빅데이터 분석 ]
	1. 빅데이터
		- 데이터베이스 관리도구의 능력을 넘어서는 대량의 정형 데이터 또는 가치를 추출할 수 있는 비정형 데이터
	2. 데이터마이닝
		- 대규모로 저장된 데이터 안에서 체계적이고 자동적으로 통계적 규칙이나 패턴을 찾아내는 기술
	3. 텍스트마이닝
		- 비정형 텍스트 데이터에서 가치와 의미가 있는 정보를 찾아내는 기술
