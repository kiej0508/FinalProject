[ 19. 설계 단계의 기획자 업무 ]
	1. 정보구조
		- AS-IS 정보구조 분석 내용과 TO-BE 개선 요구 사항에 따른 메뉴의 추가/삭제/변경사항을 반영하여 정보구조를 Depth를 순차적으로 설계
		- 설계된 정보구조는 3~4 Depth 기준의 사이트맵으로 구성되고, GNB와 LNB의 UI 설계와 연계됨로 반드시 UI의 확장성과 사용자의 접근성을 동시에 고려해야만 한다.
	
	2. 기능정의서
		- 분석 후 인사이트의 항목과 요구사항을 충족하기 위해 핈적으로 구현되어야 하는 기능 목록을 작성하고 각 기능별 세부 구현사항을 정의한다
		- 기능 정의서는 프로그래머에게는 요구사항 정의서와 같은 산출물이므로 세부 수현의 요구사항 뿐만 아니라 제약 사항 또한 함께 파악이 되어야 한다.
		- 기능정의서는 요구사항에ㅢ ID에 매칭하여 푸후에 요구사항 추적에 반영한다
	
	3. 업무흐름도
		- 업무흐름도는 서비스 단위로 사용자의 동선을 파악하여 누락되는 화면이 없도록 전사적인 흐름도를 파악함은 물론, 사용자의 흐름에 따른 화면 전대롸 함께 움직이는 데이터의 흐름도 함께 하악하기 위한 산출물
		- 화면단위의 관계는 물론 공통으로 정의되어야 하는 화면을 정의하여 화면설계서 작성 시 반영
	
	4. 화면설계서
		- 웹기획자의 최종 산출물이라고 할 수 있는 화면설계서 스토리보드는 화면 UI 설계 뿐만 아니라, 기능 정의서나 업무 흐름도에서 정의된 시스템과의 인터페이스에서 발생하는 다양한 사용자 케이스를 감안하여 사용자에게 전달하는 정보나 인터렉션 정의까지 포함하여 작성한다.


[ 20. 정보 구조도 ]
	1. 정보 구조 설계 요소
		1) 사용자
			- 사용자가 기존에 어떠한 경험과 기억을 갖고 있는지
			- 사용자는 왜 방문하고 무엇을 얻고자 하는지
			- 사용자는 이용 주엥 어떠한 패턴을 보이는지
		2) 콘텐츠
			- 컨텐츠의 중요도는 공급자 중심인지 사용자 중심인지
			- 컨텐츠를 공급자와 사용자는 어떻게 그룹핑하여 인지하는지
			- 온/오프라인에서 사용자가 인지하고 있는 네이밍은 어떠한지
		3) 시나리오
			- 기본적으로 UX를 의미
			- 기승전결의 정보의 시나리오는 어떻게 전달되어야 하는지
			- 사용자의 흐름과 함께 데이터의 흐름은 어떠한지
			- 사용자가 인지하는 비즈니스 로직은 어떠한지
	
	2. 정보 구조 설계 순서
		1) 메뉴 그룹핑
			- 채널별 메뉴 분류 및 그룹핑으로 사용자가 접근 오류를 범하지 않도록 메뉴 그룹핑 및 Depth 구조 결정
			- 채널 예시 : 상품 채널 / 소통 채널 / 개인화 채널 / 기업 채널 / 부가서비스 채널
		2) 메뉴 위치
			- UX 및 사용자 동선을 고려한 위치 결정
			- 메뉴 위치별 종류 : GNB / LNB / UTIL menu / footer / quick
		3) 메뉴 순서
			- 메뉴의 중요도와 버시스 방향과 흐름(사용자 중심/공급자 중심) 및 사용자 동선과 행동패턴을 고려하여 순서 결정
		4) 메뉴 네이밍
			- identity와 온오프라인과 연계성, 고객 커뮤니케이션 가이드를 고려한 사용자 중심의 직관적인 네이밍 결정

[ 24. UX/UI ]
	1. UX(User eXperience) : 사용자 경험
		- 사용자 경험은 사용자가 어떤 시스템, 제품, 서비스를 직, 간접적으로 이용하면서 느끼고 생각하게 되는 지각과 반응, 행동 등 총체적 경험을 말한다. 그러므로, UX 설계/디자인은 사용자 경험에 맞도록 인터렉션을 정의하고 설계하고 디자인을 해야한다는 의미로 이해
	
	2. UI(User Interface) : 사용자 매개체
		- 컴퓨터나 모바일 기계 등을 사용자가 좀 더 편리하게 사용할 수 있는 환경을 제공하는 설계 또는 그 결과물로 예를 들어, window와 같은 아이콘과 마우스를 통한 작업이 가능한 컴퓨터 인터페이스를 '그래픽 유저 인터페이스 GUI'라고 하며, 텍스트 입력으로 사용하는 인터페이스를 '캐릭터 유저 인터페이스 CUI'라고 함
		- 즉, 사용자가 시스템과 인터렉션하기 위한 모든 매개체를 말함
