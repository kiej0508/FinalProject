[ 10. 착하지만 편안한 UX, 나쁘지만 매력적인 UX ]

[ 11. 착하고 편안한 UX ]
	- 같은거인데 뭐는 한글로 뭐는 영어로
	- 중요한 기능과 덜중요한 기능의 강조 차이
	1) 직관성 : 직관적인 것 = 익숙한 것 → '익숙한 경험'
	2) 일관성
	3) 가시성
	4) Affordance : 직관성과 관련있음. 아이폰의 어플 지울 때 어플이 흔들거리듯이
	5) Fedback
	6) Depthless : 행동을 하는 Depth를 줄일수록 이탈률이 줄어들음
	→ '사용성', UX의 기본!

[ 12. 나쁘지만 매력적인 UX ]
	- 나쁘다 → 익숙하지 않다
	- 매력적이다 → 가치가 있다
	- 아이폰 : 소프트웨어 마켓이라는 엄청난 가치와 터치 인터렉션을 재정의했음

[ 13. 행동의 법칙 ]
	- COST & VALUE
	- 기본적인 행동의 원칙. 가치를 높이면 지불은 자연스럽게 이어질 것

[ Today my impression ]
편안한 UX는 기본적인 것이고, 매력적인 UX는 가치를 가진 것인데...
일단은 기본을 갖출 수 있는 사람이 되어야 하는데
그 기본에 대한 것을 그냥 대략적인 설명만 해줘서
나같은 이제 막 시작한 사람에게 길잡이가 되어주지는 못할 강의인 것 같다.
