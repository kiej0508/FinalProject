[ 07. 프로덕트/서비스의 정의와 기획자 ]
	1. Product(서비스)와 Feature(기능)
		1) Product :기능을 통해서 고객에게 가치를 줘서 기획자가 목표한 행동을 하게 하는 것
		2) Feature : 단순히 프로덕트 안에 포함된 여러가지 형태의 기술적인 기능
	
	2. Product와 Product Manager
	
	3. Product Manager의 ChecklIst
		1) 서비스의 형태와 개수
			(1) device
			(2) admin
			(3) Data/Server
		2) 정책서(or 위키)
			- 모듈 단위 어휘 정의와 정책 정의
			- 상태값과케이스 파악
			- 운영 기초 확인
		3) IA(Information Architecture)
			- Web 네비게이션을 만들 때의 정보구조
				- 페이지 단위가 명확했기 때문에 중요했지만
				- 모바일로 이동하면서는 케이스나 테스크 단위가 많아짐
			- 서비스의 page와 depth를 이해할 때!
		4) 화면설계서(스토리보드/SB) 현행본
			- 각 화면단위 기능 및 구조 확인
			- 특이사항, 예외케이스 등 정책의 구현 확인
		5) BI(Busness Intelligence)
			- 데이터 접근 권한 확인
			- 수집 가능한 데이터와 KPI확인
			- Data Science 인력과 접근 범위
			- 대표 예) 구글애널리틱스
		6) 디자인 가이드
			- 컬러 레벨과 텍스트 레벨
			- 일부 공통 UI 및 인터렉션 가이드
		7) 협업 담당자 확인
			- 기획, 디자인, 개발, 테스트 등 유관 부서와의 R&R 확인
			- 모듈 단위 핵심 질문 대상자(개발 담당자) 확인
			- 테스트 업무 R&R 관리 방식 확인
		8) 배포(DevOps/반영) 프로세스
			- 테스트 서버의 종류
			- 빌드와 배포 일정
			- 담당 부서와 긴급배포 협의자
		9) 결재/품의 프로세스
			- 각 팀간 협의 시 결제 품의 절차와 문서
			- 결재 시 유관부서 공유 등 사내 정책 확인

[ 8. 신규 서비스 기획 접근 방법]
	- 방법론 적인 내용, 실무에서 경험에 비해 공부와 업력을 필요로 하는 부분
	
	1. 신규 사업 아이디어의 발굴
		1) Megatrend 분석 > 사업 발굴
		2) Gartner's Hyper Cycle 기술 트렌드 분석 > 사업 발굴
		
	2. 신규 사업 아이디어 개발
		1) 경영전략 방식
			- 기존 사업을 기준으로 사업 포트폴리오를 확장하며 지속 가능한 사업
		2) Venture Innovation
			- VC/Startup 투자
			- Open Innovation
		3) Creative Innovation
			- UCD*(Uer Centered Design)
			- Lean Starup
			- 고객의 Pain Point 관찰, 비즈니스 모델 구축
				→ 고객의 니즈를 파악해서 서비스를 기획하는 방법
				
	3. UCD*에 기반한 사업 기획 및 개발의 과정
		1) Design Thinking (≒UCD)
			- 고객의 니즈를 판단하고 리서치로 검증을 해서 인사이트를 얻어 아이디에이션
		2) Agile
			- UCD로 아이디에이션한 것을 가설을 검정하는 과정
		3) Lean
			- 최소 단위로 개발 가능한 프로덕트로 개발, 수정을 반복?
	※ 구글 벤쳐스의 스프린트 방식

[ 9. 비즈니스 모델의 이해 ]
	1. 수익구조
		- No Market need
		- Ran out of cash
		- Not the right team
		- Get outcompeted
		- Pricing, Cost issues
		→ 수익구조가 필요하다
		
	2. 비즈니스 모델 설계
		1) 비즈니스 모델 접근법
			- Four-Box Model
			- 9 Building Blocks Model
			- STOF Model
			- RCOA Model
		2) 비즈니스 모델 공통 구성 요소
			(1) 고객 가치 제안
				- 타겟고객
				- 해결해야 할 일
				- 제공한 상품/서비스
			(2) 수익 모델 공식
				- 수익모델 : 수익창출 방식
				- 지속 가능한 이익 모델 : 이익률, 원가구조, 자원회전율, 총 투입 대비 총 수익, 성장/탈출 용인
			(3) 핵심 자원
				- 인재, 기술과 제품, 기기와 설비
				- 정보(DATA)
				- 유통경로, 파트너십 또는 제휴
				- 브랜드
			(4) 핵심 프로세스
				- 서비스 프로세스
				- 규칙과 평가 기준
				- 최저 기준
		
	3. 비즈니스 모델 캔버스(= 9 Building Blocks Model)
		- 수익의 시간 관계가 반영되기 어렵기 때문에 이런 점을 고려해서 모델을 설계할 방법을 고민해야 함
		
	4. 수익구조의 따르 접근 방법

[ 10. 서비스 프로세스와 서비스 정책의 이해 ]
	1. 서비스 프로세스란
		- 서비스가 작동되기 위해서는 '구체적인 서비스의 이용 프로세스'와 '데이터적 프로세스'가 필요
		
	2. 서비스 프로세스의 유형화와 설계
		* 도서 추천 : 비즈니스 모델 101
		1) 가치사슬 통합형
			(1) 수직 통합형 : 가치사슬 프로세스 수직 통합을 통한 가치 제고
			(2) 가치사슬 세분화 : 기존 가치사슬의 주간단계로 참여하는 방법
		2) 가치사슬 재정의형
			(1) 자산 재임대형 : 자산을 재임대해서 수익을 내는 방법
			(2) 자산 재판매형 : 가치사슬 끝난 재히를 재판매해서 수익을 내는 방법
		3) 제품 플랫폼현
			(1) 게임 콘솔형 : 디바이스 저렴하게 보급하고 컨텐츠를 지속적으로 판매하는 전략
			(2) 회원제 플랫폼형 : 메신저, 포털과 같은 강력한 회원제 기반 Open ID 고객 모객이 쉬운 라이선스 판매
		4) 가치 보장 로열티 서비스형
			(1) 구매자 보장 서비스 : 구매자 유료 회원제를 통해서 새로운 혜택을 보장
			(2) 판매자 보장 서비스 : 보유한 구매자를 보장하거나, 판매자에게 입점 자제가 메리트가 되도록 하는 서비스
	
	3. 서비스 프로세스의 정책 결정
		- 직접 시장을 분석하며 공부하는 능력을 키울 것
	
	4. 정책의 세분화 : 조직 구성과 조직 KPI
		예시 1) 영업/정산
			- 계약 진행
			- 입점사 관리
			- 상품 등록/검수
			- 정산/마감
		예시 2) 마케팅
			- 전시 관리
			- 회원 관리
			- 광고/제휴
		예시 3) CRM
			- 이메일
			- PUSH
			- 개인화추천
		예시 4) 시스템 관리
			- 실시간 로그 분석/관리
			- 트래픽 관리
		→ 큰 비즈니스 모델의 목표 속에서 서비스 운영을 위한 조직과 각 조직이 지향하는 KPI를 설정 또는 이해
		→ 세부 정책의 방향성 설정의 기준
