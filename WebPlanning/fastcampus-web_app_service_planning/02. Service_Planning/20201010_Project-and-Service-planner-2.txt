- 프로젝트 진행과 서비스 기획자 -

[ 16. 현업 요청자 ]
	1. 현업(現業)
		- 프로덕트 개발 진행 후 서비스를 운영하면서 직접 사용하거나 활용해야하는 업무자를 이르는 업계의 슬랭
		- 마케팅 담당자, 영업담당자, CS처리자 등등 프로덕트의 개선을 요청하거나 관계가 있는 모든 관리자
		- Admin 서비스의 내부 사용자에 해당
	
	2. 현업요청의 유형
		1) 업무적 목표를 위한 시스템의 의견 요청
		2) 업무 중 시스템의 불편 사항 접수
		3) 타사 서비스 캡처에 그림까지 있는 형태
		4) 와이어프레임 수준의 UI와 설명까지 있는 형태
		* 외주 기획 입장이라면 RFP(request for proposal)나 요구사항정의서의 형태로 수령
	
	3. 오퍼레이터? 기획자?
		1) Propose : 현업의 목적만을 이뤄준다면 기획자가 아니라 오퍼레이터
		2) Who & How : 기획자는 목적의 달성을 위해 누구를 위해, 어떻게 해결할 것인가를 생각하자
	
	※ 인하우스 환경에서, 현업과 기획자는 갑과 을이 아닌 협업의 관계여야 한다.
	
[ 17. 비주얼 디자이너 1 ]
	1. 디자이너에 대한 이해
		- UI디자이너 / UX디자이너 / 비주얼 디자이너 / 프로토타이퍼
		1) 비주얼 디자이너의 핵심 키워드
			- 레이아웃, 그리드, 여백, 자리수, 레벨, 통앤매너, 포인트 컬러, 룩앤필
	
	2. 디자이너 업무의 이해
		1) 드래프트
			- SB의 기본 설계 요소를 기준으로 전체적인 디자인 컨셉과 방향을 만들어보는 여러 개의 시안
		2) 기본 시안			
			- 기획자의 검토 : 의도대로 되었는지, 개발이 불가능한 디자인은 아닌지 기획자 검토 필요
			- 디자인 디렉터의 검토 : 그리드, 레이아웃 검토, UI 요소 → '운영요소'의 가이드화
		3) 시안에 대한 Variation(변화 케이스)
			- 시안의 기본 케이스를 중심으로 여러가지 케이스에 필요한 여러가지 요소를 개별 디자인 정리
			- 버큰 클릭 시, 클릭 후, 체크 시, 미체크시, 드롭다운박스 닫혔을 때, 열렸을 떄
			- 케이스 분기처리에 대한 시안 등
		4) 프로토타이핑
			- 원하는 형태의 동작과 인터렉션을 프로토타이핑 툴로 구현한 것
			- 이를 통해 디테일한 인터렉션에 필요한 디자인 세부사항을 디자인 할 수 있음
		5) 웹 스타일 가이드 작성
			- 퍼블리싱을 하기 위해 디자인에서 주는 가이드, 메인컬러, 서브컬러, 여백, 폰트, 텍스트 레벨
	
	3. 디자인 업무의 효율
		- Sketch : Variation 단축
		- Zeplin : 스타일가이드 자동 생성
		- Invision : 프로토타이팅에 Sketch 작업 파일 연동
	
	4. 디자이너와의 커뮤니케이션
		1) 비즈니스와 기획적 관점에서 의논, 취향으로 접근하지 말 것
		2) 기획자는 컨펌자가 아닌, 의논하고 논의하는 협업자
		3) 모바일의 가변적인 환경을 이해하고 시안을 검토, 또 다른 협업자들의 입장에 대해서도 충분히 대변
	
[ 18. 개발자 ]
	1. 개발자에 대한 이해
		1) SW형상 관리 빌드 담당자 : 깃허브, SVN 등의 형상관리를 담당하고 소스 빌드 및 배포하는 담당자
		2) TA(Technical Architecture) : 데이터, 애플리케이션 아키텍처를 지원하는 데 필요한 정보기술 인프라 요소 및 구조, 그리고 이들 간의 관계를 정의하는 아키텍처 담당자
		3) DBA(Database Administrator) : 데이터베이스의 테이블 구조 및 인덱스, 쿼리 검수 등 관리자. AWS의 보급으로 점점 더 Data Science 영역의 업무가 요구됨
		4) 시스템 프로그래머 : 대형 시스템의 내부에서 OS처럼 움직일 시스템을 설계하는 개발자(국내 많이 없음)
		5) 보안 프로그래머 : 정보보호와 방화벽, 네트워크 보안 등을 담당하는 프로그래머
		6) 웹 프로그래머 : JAVA, C++, python 등의 언어로 웹 어플리케이션을 개발하는 프로그래머
		7) 임베디드 프로그래머 : IoT, 스마트폰 등 디바이스에 이식된 소프트웨어를 개발하는 프로그래머
	
	2. 개발자와의 커뮤니케이션
		1) 용어의 문제
			- 개발 언어를 배우는 것이 도움이 될까
			- 자사의 용어사전을 만들어서 어휘를 통일하자
		2) 무시의 문제
			- 개발, 시스템에 잘 모르는 것을 두려워 말자
			- 기획자는 비즈니스와 UX로 대화해야 한다
